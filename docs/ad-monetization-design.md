# 広告マネタイズ設計書 - Workforce Nexus

## 概要

ユーザーに無料でサービスを提供しつつ、広告収益でビジネスを維持する「広告支援型フリーミアム」モデルの設計。

---

## 1. プラン構成

| プラン | 料金 | 広告 | 機能 |
|--------|------|------|------|
| **FREE** (広告あり) | 無料 | 表示あり | 全機能利用可 |
| **PREMIUM** (広告なし) | 有料 (Stripe課金) | 非表示 | 全機能 + 優先サポート |

### 移行フロー

```
新規登録 → FREE (広告あり、機能制限なし)
         ↓ アップグレード
       PREMIUM (広告なし、Stripe課金)
         ↓ 支払い失敗
       SUSPENDED (利用停止)
         ↓ ダウングレード選択
       FREE (広告ありに戻る)
```

**ポイント**: トライアル期間を廃止し、最初から全機能を無料で使える代わりに広告を表示。
有料プランは「広告非表示」が最大の訴求ポイント。

---

## 2. 広告配置戦略

### 2.1 配置原則

- **打刻操作を絶対に妨げない**: 出勤・退勤ボタン周辺には広告を配置しない
- **業務効率を下げない**: インタースティシャル広告は使わない
- **プロフェッショナルな見た目**: B2B SaaSとしての品位を保つ
- **日本市場に最適化**: 日本企業向けの広告ネットワークを使用

### 2.2 広告スロット定義

| スロットID | 配置場所 | フォーマット | サイズ | 表示頻度 |
|-----------|---------|-------------|--------|---------|
| `dashboard-top` | ダッシュボード上部 | バナー | 728x90 (PC) / 320x50 (SP) | 常時 |
| `dashboard-bottom` | ダッシュボード下部 | レクタングル | 300x250 | 常時 |
| `admin-sidebar` | 管理画面サイドバー | レクタングル | 300x250 | 常時 |
| `daily-reports-footer` | 日報一覧下部 | バナー | 728x90 | 常時 |
| `post-clockout` | 退勤完了後 | インフィード | 468x60 | 退勤時のみ |

### 2.3 ページ別配置マップ

```
┌─────────────────────────────────────────┐
│  ヘッダー (ロゴ, ユーザー情報)            │
├─────────────────────────────────────────┤
│  [AD] dashboard-top (バナー 728x90)     │  ← 控えめなバナー
├─────────────────────────────────────────┤
│  今日の打刻情報                           │
│  ┌─────────────────────────────────┐    │
│  │  出勤 / 休憩 / 退勤 ボタン       │    │  ← 広告なし (業務最優先)
│  └─────────────────────────────────┘    │
│                                         │
│  日報パネル                              │
│  打刻履歴                                │
│  休暇情報                                │
│                                         │
│  ┌───────────────┐                      │
│  │ [AD] 300x250  │  ← dashboard-bottom  │
│  │ レクタングル    │                      │
│  └───────────────┘                      │
│                                         │
│  [Premium へのアップグレード誘導]          │  ← 広告非表示の訴求
└─────────────────────────────────────────┘
```

## 3. 広告プロバイダー

### 3.1 推奨構成

| 優先度 | プロバイダー | 用途 |
|--------|-------------|------|
| 1 | **Google AdSense** | メインの広告配信 (日本市場カバー率最高) |
| 2 | **A8.net アフィリエイト** | HR/ビジネスサービス系の成果報酬型 |
| 3 | **自社広告** | プレミアムプランへのアップグレード訴求 |

### 3.2 ターゲティングカテゴリ

勤怠管理SaaSのユーザー層に適した広告カテゴリ:
- オフィス用品・備品
- ビジネスSaaS (会計、CRM、プロジェクト管理)
- 人材採用・求人サービス
- ビジネス保険
- 企業研修・資格取得
- クラウドサービス

## 4. 技術設計

### 4.1 コンポーネント構成

```
src/
├── components/
│   └── ads/
│       ├── AdBanner.tsx        # バナー広告コンポーネント
│       ├── AdRectangle.tsx     # レクタングル広告コンポーネント
│       ├── AdSlot.tsx          # 汎用広告スロット (プラン判定込み)
│       └── PremiumUpsell.tsx   # プレミアムアップグレード誘導
├── lib/
│   └── ads.ts                  # 広告設定・スロット定義・ヘルパー
```

### 4.2 広告表示フロー

```
1. ページ読み込み
2. セッションからテナントのプランを取得
3. plan === "FREE" の場合のみ AdSlot を描画
4. AdSlot が Google AdSense スクリプトをロード
5. 広告が表示される
6. インプレッション/クリックを計測 (Google側で自動)
```

### 4.3 プラン判定

```typescript
// AdSlot コンポーネント内
if (tenant.plan === "PREMIUM") {
  return null; // 有料プランでは広告非表示
}
return <div className="ad-slot">...</div>;
```

### 4.4 環境変数

```env
# Google AdSense
NEXT_PUBLIC_ADSENSE_CLIENT_ID=ca-pub-XXXXXXXXXX
NEXT_PUBLIC_ADSENSE_SLOT_DASHBOARD_TOP=1234567890
NEXT_PUBLIC_ADSENSE_SLOT_DASHBOARD_BOTTOM=1234567891
NEXT_PUBLIC_ADSENSE_SLOT_ADMIN_SIDEBAR=1234567892
NEXT_PUBLIC_ADSENSE_SLOT_REPORTS_FOOTER=1234567893
```

## 5. 収益試算

### 5.1 前提条件

- 無料ユーザー: 1,000テナント x 平均15ユーザー = 15,000 DAU
- 1ユーザーあたり平均 PV/日: 3 (ダッシュボード + 1-2ページ)
- 広告表示回数/日: 45,000 imp
- eCPM (日本市場 B2B): ¥200〜500

### 5.2 月間収益見込み

| シナリオ | eCPM | 月間imp | 月間収益 |
|---------|------|---------|---------|
| 控えめ | ¥200 | 1,350,000 | ¥270,000 |
| 標準 | ¥350 | 1,350,000 | ¥472,500 |
| 楽観 | ¥500 | 1,350,000 | ¥675,000 |

### 5.3 ハイブリッド収益 (広告 + Premium課金)

Premium転換率 10% の場合:
- 広告収益: ¥425,250/月 (90%が無料ユーザー)
- Premium課金: 100テナント x ¥5,000/月 = ¥500,000/月
- **合計: 約¥925,250/月**

## 6. UX配慮事項

### 6.1 広告の制約ルール

1. **1ページあたり最大2枠**: 広告過多によるUX低下を防止
2. **打刻ボタンから100px以上離す**: 誤タップ防止
3. **Cumulative Layout Shift (CLS) 対策**: 広告枠を事前にリザーブ
4. **ダークモード対応**: 広告枠の背景をテーマに合わせる
5. **モバイル最適化**: 320x50の小型バナーを使用

### 6.2 Premium アップグレード訴求

- 月に1回、ダッシュボード下部に控えめな誘導バナー
- 広告の「この広告を非表示にする」リンクからアップグレードページへ
- 管理者向け billing ページでの比較表
